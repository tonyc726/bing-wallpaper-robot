## 🧭 关键决策(含理由)

### 1. 存储:ImageKit → Cloudflare R2(仅作备份)
- **发现**:前端图片 URL 直接走 Bing CDN(`makePreviewJSON.ts:269-270`),ImageKit 在本系统里其实是「UHD 原图异地备份仓库」,不是分发 CDN。撞的是**存储墙**(3GB/账号,两账号约 80%),非带宽墙。
- **决策**:迁 R2(10GB 免费、egress 免费、$0.015/GB 超额、与 Cloudflare Pages 同生态)。
- **状态**:🔴 阻塞于用户准备 R2 账户。

### 2. 否决 TinyPNG 前置压缩
- **理由(Inversion)**:① 有损压缩会污染唯一备份,Bing 原图未来失效后原画质永久丢失;② 前端不读备份图,压缩对加载零收益;③ ROI 为负且引入 500 张/月额度墙 + 外部依赖。
- **替代**:改用**本地 `sharp`**(免费无额度、AVIF 压缩率更高)。

### 3. 双版本备份(用户确认)
- R2 存 `{id}.jpg`(无损原图,对冲 Bing 失效)+ `{id}.avif`(sharp 生成的优化版,为将来自主分发预备)。

### 4. 暂缓 P0-1 多市场采集
- **二阶思维**:多市场会让每日新增原图翻几倍,而 ImageKit 已 80% 满、R2 未就绪 → **先迁 R2 再扩市场**,否则加速撞墙。

### 代码改动要点
- **P1-1 下载防御**(`download-image.ts` / `download-thumbnail.ts`):校验 HTTP 状态码(404 占位页不再入库)、30s 超时、`req.on('error')` 兜底、失败清理半截文件、`settled` 防重复 resolve/reject。
- **P1-2 SSIM 本地优先**(`add-or-update-wallpaper.ts`):抽出 `confirmSimilarityBySSIM`,优先读本地 `docs/thumbs/${id}.${ext}`,缺失才回退下载,消除对 Bing 的重复请求。
- **顺带修复**:抽出 `isColorHistBeyondThreshold`,修掉一个既有的 `max-depth`(>5)ESLint 违规。

## 🔍 会话中确认的事实(curl 实测,2026-07-03)

- Bing `HPImageArchive.aspx` 的 `mkt` 参数有效,**同日不同市场是不同壁纸**;加 `uhd=1&uhdwidth=3840&uhdheight=2160` 可取 4K 原图 URL。
- 非官方 `bing.com/hp/api/v1/imagegallery?format=json&mkt=xx` 返回长文 description + 竖屏图变体(⚠️ 无文档,需 fallback)。
- `bing.npanuhin.me` 历史归档 2010 年起可用,可回填缺口。
